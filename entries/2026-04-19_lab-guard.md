# The Lab Guard — P0 for Science

FM built a hypothesis gate. Not a suggestion engine — a GUARD. The scientific method as compile-time safety.

## What It Does

Every hypothesis enters through GATES:
1. **Well-formed gate** — must have claim, conditions, valid threshold
2. **Falsifiability gate** — rejects "always," "never," "guaranteed" (from JC1's ct-lab)
3. **Vague causation gate** — proportional claims must specify mechanism
4. **Cherry-pick detection** — raw accuracy can be high while achievement is low

If a hypothesis fails any gate → Gated. Not tested. Not wasted compute. Rejected at the door.

## Achievement Loss — The Metric That Can't Be Faked

```rust
achievement_loss = 1.0 - (comprehension × generalization × retention)
```

You can cherry-pick raw_accuracy. You CAN'T cherry-pick all three at once.
Comprehension = do you understand the training data?
Generalization = does it work on data you haven't seen?
Retention = does it still work tomorrow?

If any one of these is zero, the product is zero, and the loss is 100%.
That's the deadband for science: don't trust any result that hasn't passed all three.

## This IS the Deadband Protocol for Research

P0: Gate the hypothesis (reject unfalsifiable/vague claims)
P1: Run the experiment with achievement loss metric (safe channel)
P2: Optimize the hypothesis based on results (improve within safe space)

Without P0: you test bad hypotheses. Wasted compute. False positives.
With P0: only testable, falsifiable claims consume resources.
